# agents-multiagent

Multiagent configuration for Claude Code and API-based workflows.
Implements a configurable agent pipeline with structured JSON handoffs.

## Structure

```
agents-multiagent/          # this repo, added as submodule at agents/
  AGENTS.md              # Rules + workflow loader (never modify)
  workflow.md            # Default pipeline: Architect → Coder → Reviewer
  architect.md           # Architect role definition
  coder.md               # Coder role definition
  reviewer.md            # Reviewer role definition
  schemas/
    architect-output.json
    coder-output.json
    reviewer-output.json
  extensions/
    role-template.md     # Template for creating new roles (copy to your project)

your-project/               # your repository
  agents/                # submodule → agents-multiagent
  agents-extensions/     # your customisations (outside the submodule)
    workflow.md          # optional pipeline overlay
```

---

## How it works

`AGENTS.md` contains immutable code rules. It does not define the pipeline.
The pipeline is resolved by merging two files:

- `agents/workflow.md` — always loaded as the default pipeline
- `agents-extensions/workflow.md` — optional overlay, merged on top

When the overlay exists, sections defined in it override the matching sections
from the default. Sections absent from the overlay remain active from the
default unchanged. The entire default is never replaced.

## Setup

### Add as Git submodule

```bash
git submodule add git@github.com:janmarkuslanger/agents-multiagent.git agents
```

Claude Code picks up `agents/AGENTS.md` automatically.

On a fresh clone:

```bash
git clone --recurse-submodules git@github.com:USERNAME/my-project.git
```

Or after cloning without submodules:

```bash
git submodule update --init --recursive
```

Update to the latest version:

```bash
git submodule update --remote agents
git add agents && git commit -m "chore: update agents"
```

## Default workflow

```
1. ARCHITECT  →  design document (JSON) → wait for confirmation
2. CODER      →  implementation (JSON)
3. REVIEWER   →  verdict (JSON)
```

Defined in `agents/workflow.md`. Do not edit this file — override it instead.

## Extending the workflow

### Option A: Extend the pipeline (workflow extension)

`agents/workflow.md` is always loaded first as the base. If
`agents-extensions/workflow.md` exists in your project, it is loaded on top.
**Sections defined in the extension override the matching base sections.**
Everything else stays unchanged from the base.

This means your extension only needs to declare what actually changes —
no need to repeat boilerplate like the Architect skip question or the
iteration rule.

```bash
mkdir agents-extensions
touch agents-extensions/workflow.md
git add agents-extensions/workflow.md
git commit -m "chore: add workflow extension"
```

**Example — inserting a Platform Engineer role:**

Only `## Roles`, `## Pipeline`, and `## Handoff` change. Everything else
(Architect skip question, iteration, handoff format rules) stays from the base.

````markdown
## Roles

| Role               | File                       | When to activate                            |
|--------------------|----------------------------|---------------------------------------------|
| Software Architect | `agents/architect.md`      | New feature or boundary change              |
| Coder              | `agents/coder.md`          | After confirmed design document             |
| Platform Engineer  | `platform-engineer.md`     | After Coder — self-skips if no infra impact |
| Reviewer           | `agents/reviewer.md`       | After Platform Engineer                     |

## Pipeline

```text
1. ARCHITECT          →  design document (JSON) → wait for confirmation
2. CODER              →  implementation (JSON)
3. PLATFORM ENGINEER  →  infra + deployment (JSON), or { "skipped": true }
4. REVIEWER           →  verdict (JSON)
```

## Handoff

- Coder receives: Architect's confirmed design document
- Platform Engineer receives: design document + Coder's implementation
- Reviewer receives: design document + Coder's implementation +
  Platform Engineer's output (or skip marker)

All handoffs are inline JSON. No files are written to disk.
````

That's the entire extension file. The Architect skip question, iteration rules,
and mandatory workflow note all come from the base automatically.

This extension requires a `platform-engineer.md` role file in your project.
See Option B for how to create one.

### Option B: Create a new role

Copy the template and fill it in:

```bash
cp agents/extensions/role-template.md platform-engineer.md
```

The template covers: responsibilities, inputs, skip condition, output schema
reference, and constraints. A role file answers these questions:

- What is this role responsible for?
- What does it receive as input (which upstream role outputs)?
- Under what condition does it self-skip?
- What JSON does it output?

**Example output schema** for an optional role (one that can skip itself):

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "oneOf": [
    {
      "type": "object",
      "properties": {
        "skipped": { "type": "boolean", "enum": [true] },
        "reason":  { "type": "string" }
      },
      "required": ["skipped", "reason"],
      "additionalProperties": false
    },
    {
      "type": "object",
      "properties": {
        "summary":     { "type": "string" },
        "infra_changes": { "type": "array", "items": { "type": "string" } },
        "risks":       { "type": "array", "items": { "type": "string" } }
      },
      "required": ["summary", "infra_changes", "risks"],
      "additionalProperties": false
    }
  ]
}
```

Reference the schema file in the role's `## Output` section.

### Option C: Append rules to an existing role

For project-specific rules that extend a role without replacing it
(framework conventions, naming rules, stack constraints):

**Via API:**

```python
base  = Path("agents/coder.md").read_text()
extra = Path("my-coder-extension.md").read_text()
system = base + "\n\n---\n\n" + extra
```

**Via Claude Code:** reference the extension in your project's root `AGENTS.md`:

```markdown
## Role extensions

The Coder appends rules from `extensions/coder-django.md`.
```

## Design rules for workflow extensions

- Only define sections that actually change — omit everything else
- If you override `## Roles`, include the full updated table (not just new rows)
- If you override `## Pipeline`, include all steps in order
- Every optional role must self-skip with `{ "skipped": true, "reason": "..." }`
- Do not repeat rules from the base that remain unchanged (Architect skip
  question, handoff format, iteration rule, mandatory workflow note)
